# Planner Pay

Веб-сервис напоминаний об оплатах. Многопользовательский, с изолированными
кабинетами, дашбордом в стиле GitHub-активности, подсчётом расходов по месяцам
и автоматической рассылкой напоминаний по SMTP **за 14 дней** и **за 1 день**
до даты оплаты. Интерфейс в стиле macOS, светлая и тёмная темы.

## Возможности

- **Регистрация с подтверждением администратором** — новые заявки активируете вы
  (на дашборде админа показывается баннер с числом заявок).
- **Изолированные кабинеты** — каждый пользователь видит только свои оплаты.
- **Поля оплаты**: дата, сервис, сумма, описание (дату нельзя ставить задним числом).
- **Напоминания по почте** (SMTP) за 2 недели и за 1 день. У каждого пользователя
  можно указать **несколько** адресов — письмо уходит на все.
- **Ручная отправка** письма-сводки за выбранный период (7/14/30/90 дней или все).
- **Продление** — у оплаченной записи выбираете дату и сумму следующего платежа;
  старая помечается оплаченной (копится история).
- **Дашборд в стиле GitHub** — тепловая карта по дням (неделя Пн–Вс), при наведении
  раскрывается список оплат на дату; столбчатый график расходов по месяцам;
  суммы за год (всего / оплачено / предстоит).
- Светлая и тёмная темы.

## Стек

Python · FastAPI · SQLite (SQLAlchemy) · Jinja2 · APScheduler · Docker Compose.
Все секреты — в `.env`.

## Запуск

```bash
cp .env.example .env        # заполните SECRET_KEY, ADMIN_*, SMTP_*
docker compose up --build
```

Откройте <http://localhost:8000>.

Войдите администратором (`ADMIN_USERNAME` / `ADMIN_PASSWORD` из `.env`).
Новые пользователи регистрируются на `/register`, а вы подтверждаете их
в разделе **Админ**.

База данных — обычный файл **`./data/planner_pay.db`** (бинд-маунт), его легко
бэкапить и переносить.

> Без настроенного SMTP письма не отправляются, а пишутся в лог контейнера —
> удобно для локальной разработки. Проверить отправку: **Настройки → Отправить
> тестовое письмо**.

## Локальный запуск без Docker

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r app/requirements.txt
export $(grep -v '^#' .env | xargs)              # подхватить переменные
export DATABASE_URL="sqlite:///./data/planner_pay.db"
uvicorn app.main:app --reload                    # из корня репозитория
```

## Как работают напоминания

Планировщик (APScheduler) ежедневно в `REMINDER_HOUR:REMINDER_MINUTE`
(часовой пояс `TIMEZONE`) проверяет все неоплаченные записи и отправляет письмо,
если до даты осталось ровно 14 или 1 день. Флаги в БД исключают повторную отправку.

## Структура

```
app/
  main.py            точка входа, инициализация БД и админа, маршруты
  config.py          настройки из .env
  database.py        подключение к SQLite
  models.py          модели User и Payment
  security.py        пароли, сессии, проверки доступа
  email_utils.py     отправка писем по SMTP + шаблон письма
  reminders.py       логика рассылки и продления
  scheduler.py       ежедневная фоновая задача
  routers/           auth, dashboard, payments, admin, settings
  templates/         Jinja2-шаблоны
  static/            CSS и JS (тепловая карта, тема, модалки)
```
